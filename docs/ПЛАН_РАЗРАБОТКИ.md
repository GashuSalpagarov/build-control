# План разработки - Контроль строительства

## Принципы
- Простота - минимум абстракций
- Масштабируемость - tenantId для мультитенантности
- Надёжность - PostgreSQL, стандартные решения

---

## Технологический стек (утверждено)

### Frontend
- Next.js 14 (PWA, App Router)
- TypeScript
- TailwindCSS + shadcn/ui
- React Hook Form + Zod
- Recharts (графики)
- Таблица-календарь на CSS Grid (без библиотеки Gantt)

### Backend
- NestJS
- Prisma ORM
- JWT авторизация
- Multer (загрузка файлов)

### Инфраструктура
- PostgreSQL
- Папка на диске для файлов (с абстракцией для будущего перехода на S3)
- Docker Compose
- Nginx

### Что НЕ используем на старте
- Redis
- Очереди задач (Bull)
- Monorepo (Turborepo)
- Библиотеки Gantt

## Роли пользователей

| Роль | Описание | Доступ |
|------|----------|--------|
| **Суперадмин** | Управление tenants | Всё |
| **Правительство** | Read-only ко всему | Только просмотр |
| **Министр** | Полный доступ в рамках tenant | Всё кроме tenants |
| **Технадзор** | План-график, проверки объёмов | Объекты, этапы, проверки объёмов, обращения |
| **Бухгалтер** | Платежи | Только свои объекты, только финансы |
| **Проверяющий** | Ежедневные проверки ресурсов | Только форма проверки, НЕ видит план |
| **Подрядчик** | Просмотр своих объектов | Свои объекты, обращения |

## Основные сущности БД

### Tenant
- id, name, slug

### User
- id, tenantId, role, email, name, phone, passwordHash, isActive

### UserObjectAssignment (привязка пользователя к объектам)
- id, userId, objectId

### Contractor (подрядчик)
- id, tenantId, name, inn, phone, email

### Object (объект строительства)
- id, tenantId, contractorId, name, address, startDate, endDate, budget, status

### Stage (этап работ)
- id, objectId, name, startDate, endDate, budget, plannedPeople, sortOrder

### EquipmentType (справочник техники)
- id, tenantId, name

### PlannedEquipment (плановая техника на этап)
- id, stageId, equipmentTypeId, quantity

### ResourceCheck (ежедневная проверка ресурсов)
- id, stageId, userId, date, actualPeople, comment
- unique: (stageId, date)

### EquipmentCheck (факт техники в проверке)
- id, resourceCheckId, equipmentTypeId, quantity

### VolumeCheck (проверка объёмов)
- id, stageId, userId, date, percent, comment

### Payment (платёж)
- id, stageId, userId, date, amount, paymentNumber, comment

### Appeal (обращение)
- id, objectId, stageId, userId, type, subject, description, status, createdAt

### AppealMessage (сообщение в обращении)
- id, appealId, userId, text, createdAt

### Attachment (прикреплённые файлы)
- id, appealId, resourceCheckId, path, filename, createdAt
